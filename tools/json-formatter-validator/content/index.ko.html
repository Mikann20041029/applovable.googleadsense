Quick answer

JSON이 실패하는 가장 흔한 원인은 후행 쉼표, 키 따옴표 누락, 작은따옴표 사용, 괄호 불일치입니다. 왼쪽에 붙여넣고 검증을 눌러 오류(가능하면 줄/열)를 확인한 뒤, 정렬(가독성) 또는 축소(용량 감소)를 사용하세요.

사용 방법

  1. 입력창에 JSON을 붙여넣습니다. 필요하면 예시로 정상 JSON을 불러옵니다.
  2. 검증을 눌러 strict JSON 문법을 확인합니다. 오류가 있으면 메시지와 위치를 표시합니다.
  3. 정렬로 보기 좋게, 축소로 공백을 제거해 전송용으로 만듭니다.
  4. 키 정렬은 텍스트 비교가 중요한 시스템에서는 주의해서 사용합니다.
  5. 결과는 복사/다운로드할 수 있고, 입력은 브라우저에 로컬 저장됩니다.

개인정보: 모든 처리는 브라우저에서만 수행되며 입력은 서버로 전송되지 않습니다.

유효한 JSON의 규칙

  • 키는 반드시 큰따옴표: {"name":"A"}
  • 문자열도 큰따옴표만 허용
  • 후행 쉼표 금지: {"a":1,} 는 실패
  • 주석 금지: //, /* */ 불가

Examples

Input: {"a":1,"b":2}
Output:
{
  "a": 1,
  "b": 2
}

Troubleshooting

  • 후행 쉼표 제거
  • 작은따옴표 → 큰따옴표
  • 키 따옴표 추가
  • 주석 제거
  • 백슬래시 이스케이프(\\)
  • 문자열 내 줄바꿈은 \n
  • 괄호 닫힘 확인
  • NaN/Infinity 사용 금지
  • UTF-8 저장 권장
  • 잘린 JSON 복구
