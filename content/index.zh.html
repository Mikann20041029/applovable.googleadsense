Quick answer

JSON 最常见的失败原因是：末尾多余逗号、对象键未加双引号、使用单引号、括号不匹配。 把 JSON 粘贴到左侧，点击校验查看错误（可能含行/列），再用格式化或压缩输出最终结果。

使用步骤

  1. 粘贴 JSON，必要时点示例加载正确样例。
  2. 点击校验，检查 strict JSON 语法。
  3. 点击格式化生成易读缩进，或压缩去除空白用于传输。
  4. 键排序会改变原始文本顺序，只有在确定安全时使用。
  5. 输出可复制/下载，输入会保存在浏览器本地。

隐私：全部在浏览器本地运行，输入不会发送到服务器。

有效 JSON 的关键规则

  • 键必须使用双引号：{"name":"A"}
  • 字符串只能用双引号
  • 禁止末尾逗号：{"a":1,} 无效
  • 禁止注释：//、/* */ 不支持

排错要点

  • 删除末尾逗号
  • 单引号改双引号
  • 补齐括号与引号
  • 反斜杠需要转义（\\）
  • 字符串内换行用 \n
